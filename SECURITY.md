# Security

Thanks for helping keep this project and its users safe.

We take the security of our software products and services seriously, including all open source code repositories managed under this organization.

Please note that this repository is outside the scope of any bug bounty program, but we will ensure that your finding is passed along to the appropriate maintainers for remediation.

## Reporting Security Issues

If you believe you have found a security vulnerability in any repository owned by this project, please report it to us through coordinated disclosure.

**Please do not report security vulnerabilities through public issues, discussions, or pull requests.**

Instead, please send an email to the project maintainers at [provide a secure contact email, e.g., security@example.com].

Please include as much of the information listed below as you can to help us better understand and resolve the issue:

- The type of issue (e.g., buffer overflow, SQL injection, or cross-site scripting)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit the issue

This information will help us triage your report more quickly.

## Policy

We follow standard coordinated disclosure practices. For more details, please contact the maintainers directly.
